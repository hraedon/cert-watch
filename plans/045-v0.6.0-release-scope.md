# Plan 045 — v0.6.0 Release Scope

> **Status:** delivered 2026-06-06 (pending merge to `main` + `v0.6.0` tag).
> Umbrella plan for the 0.6.0 release. The headline-feature design lives in
> [Plan 039 (API-key auth)](039-api-key-auth.md).

## Context

A substantial body of work had accumulated since `v0.5.3` (RBAC wired,
api.py decomposition, mypy→0, a dashboard perf fix) with the version already
bumped to 0.6.0 but uncommitted, no CHANGELOG entry, and the E2E suite covering
only a subset of pages on brittle selectors. 0.6.0 **locks down RBAC + adds
machine-to-machine automation, proven by a full E2E UI regression suite**, and
gates quality in CI so the behaviour can't silently regress.

User decisions: net-new feature = **API keys (Plan 039) only**; E2E =
**functional + visual regression** on stable `data-testid` selectors.

## Theme

Make the release the version bump already implied: everything since 0.5.3 is
committed, documented, tested through the browser, and CI-gated — plus scoped
API keys.

## Workstreams (all delivered)

- **A — Working-tree closeout.** Verified the BC-139 dashboard grouped-path
  rewrite (added a kinds-coexistence test); committed the mypy cleanup + config
  and the dashboard fix in coherent chunks. Version re-bump deferred to F.
- **B — API keys (Plan 039 / BC-104).** `m0015` table + `SqliteApiKeyRepository`
  (SHA-256-hashed `cwk_…` tokens, scopes read/write/admin); bearer auth wired
  into `auth_middleware` + `require_auth`; `require_admin`/`require_admin_write`;
  `/api/api-keys` CRUD + `/settings/api-keys` UI; audited; 16 tests.
- **C — Full E2E suite (BC-132/BC-145).** C1: `data-testid` across all templates
  + refactored existing selectors. C2: functional coverage of every page/flow.
  C3: RBAC admin-vs-viewer gating asserted in the browser (crafted cookies,
  pinned signing key — no IdP needed). C4: masked visual baselines under
  `tests/e2e/__screenshots__/`, `-m visual`. 35 functional + 8 visual.
- **D — CI gates.** mypy gated in `ci.yml`; `e2e.yml` split into required
  functional job + non-blocking visual job (uploads diff artifacts). Template
  CSP budgets already gated by existing unit tests (BC-143 stays deferred).
- **E — UI breadcrumbs.** BC-130 alert chips reflect configured channels
  (fixed + tests). BC-128 (cert-detail owner/renewal/runbook edit) was found
  already implemented and working — added testids. BC-129 deferred.
- **F — Release.** CHANGELOG `[0.6.0]` (with the RBAC upgrade note) + `[0.5.3]`;
  RBAC + API-key docs in README (BC-148); version re-bumped to 0.6.0; merge to
  `main` and tag `v0.6.0` (release.yml builds/scans the image).

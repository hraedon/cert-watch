## Problem

`auth/ldap_provider.py:152-157` builds the required-groups filter with the
hardcoded AD matching-rule OID `1.2.840.113556.1.4.1941`
(`LDAP_MATCHING_RULE_IN_CHAIN`, AD transitive membership):

```python
f"(memberOf:1.2.840.113556.1.4.1941:={escape_filter_chars(g)})"
```

This OID is **AD-specific**. On OpenLDAP, FreeIPA, or other directories it is
unsupported, so `LDAP_REQUIRED_GROUPS` checks silently fail and valid users are
denied. The README advertises generic "LDAP/AD" auth, so this is a real
portability gap, not just a missing nicety.

## Suggested fix

Make the group-membership filter configurable, e.g. `LDAP_GROUP_FILTER` with a
`{group}` placeholder, defaulting to the current AD transitive form for
backward-compat. Document common non-AD forms (plain `(memberOf={group})`, or
`(member={user_dn})` for posixGroup/groupOfNames). At minimum, document the
current AD-only limitation. A directory-shaped unit test (mocked `ldap3`) should
cover both the AD-transitive and a plain-`memberOf` filter.

Scoped for 0.5.0 — see Plan 023.

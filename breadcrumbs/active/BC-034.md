# BC-034: Owner/contact field for certificates and hosts with alert routing

**Type:** feature
**Severity:** medium
**Status:** new
**Filed:** 2026-05-29

## Problem

When a certificate alert fires, the most important information isn't "cert expires in 3 days" — it's "who owns this and how do I reach them." Currently, the only way to associate ownership is via the freeform `notes` field, which is unstructured and not queryable.

For teams managing more than a handful of certificates, the workflow is:
1. Alert fires to a central inbox or Slack channel.
2. Someone looks at the cert details.
3. They try to figure out who owns it (ask around, check a wiki, grep a config repo).
4. They contact the owner.

Steps 2-3 are wasted time that happens every single time.

## Suggested fix

Add structured owner/contact fields:

### Schema
- **`owner_name`** (text, optional) — display name (e.g., "Jane Doe", "Platform Team")
- **`owner_email`** (text, optional) — email address for alert delivery
- **`owner_slack`** (text, optional) — Slack handle or channel (e.g., "@jane.doe", "#platform-oncall")

These fields belong on the `hosts` table (per host, not per cert), because ownership is a deployment concern, not a cert property. The same cert deployed by two different teams has two different owners.

### Alert integration
- When an alert fires, include the owner info in the message:
  - Email: "Owner: Jane Doe (jane@example.com)"
  - Webhook: include `owner_name`, `owner_email`, `owner_slack` fields in the payload
- When `owner_email` is set, send the alert to the owner **in addition to** the global `ALERT_RECIPIENTS`. This enables per-team alert routing without replacing the central notification.

### Renewal status
Add a `renewal_status` field to the `hosts` table:
- `pending` — default, no action taken
- `in_progress` — someone is working on the renewal
- `renewed` — cert has been renewed and deployed

When `renewal_status` is `in_progress`, the alert system downgrades alerts to informational (no urgency escalation). When `renewed`, alerts are suppressed until the next expiry window.

This prevents duplicate alert noise ("yes, I know, I'm working on it") and gives ops confidence that someone is handling it.

### UI
- Show owner name on the dashboard row (collapsed view).
- Show full owner details on hover or in the expanded view.
- Add a dropdown or button to change `renewal_status` from the dashboard.
- Filter by owner in the dashboard query.

## Problem

The backend now has `chain_incomplete` on `ScannedEntry` (added in response to Perplexity feedback), but the UI posture panel does not show it. When a Python < 3.13 host without openssl falls back to leaf-only scanning, the posture grade already drops (`chain_status == "incomplete"` → grade penalty), but the user has no visibility into **why** the chain is incomplete.

## Suggested fix

Add a warning badge or tooltip on the certificate detail page explaining:

> "Chain incomplete: scan ran with degraded certificate extraction. Consider upgrading to Python 3.13 or ensuring openssl is available."

This is a follow-up to the backend signal that is currently invisible.

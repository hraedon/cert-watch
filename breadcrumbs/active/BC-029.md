# BC-029: REST API pagination lacks HATEOAS navigation links

**Type:** design-question
**Severity:** low
**Status:** new
**Filed:** 2026-05-28

## Problem

The REST API endpoints (`/api/certificates`, `/api/hosts`, `/api/alerts`) return pagination metadata (`page`, `limit`, `total`, `pages`) but do not include `_next`, `_prev`, or `_self` URLs. Clients must construct pagination URLs manually by understanding the query parameter conventions.

This makes the API harder to consume for automation clients and doesn't follow HATEOAS principles. It's also inconsistent with the `/metrics` and `/healthz` endpoints which are fully self-describing.

## Suggested fix

Add `links` to the pagination response object:

```json
{
  "pagination": {
    "page": 1,
    "limit": 50,
    "total": 120,
    "pages": 3,
    "next": "/api/certificates?page=2&limit=50",
    "prev": null,
    "self": "/api/certificates?page=1&limit=50"
  }
}
```

The base URL can be derived from the request. This is a non-breaking additive change.

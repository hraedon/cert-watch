# BC-043: E2E suite bypasses /setup wizard instead of testing it

**Type:** todo
**Severity:** medium
**Status:** new
**Filed:** 2026-06-01

## Problem

The E2E `conftest.py` sets `CERT_WATCH_ALLOW_UNAUTH=1` to suppress the setup redirect, which keeps the existing tests green but means no automated test exercises the `/setup` wizard flow. The wizard is the first-run experience for every new user; if it breaks, users cannot even reach the dashboard. This is a critical first-impression path with zero coverage.

## Suggested fix

Add a dedicated E2E test that goes through the wizard: create local admin → log in → reach dashboard. The existing tests can keep the bypass for their own scenarios, but a new test file (`test_setup_wizard.py`) should cover the full wizard journey.

## Context

The adversarial review fix changed 16 source files but added no new tests. The existing 2330-test suite passes, but the new code paths are not exercised by existing tests. Specific gaps:

1. **scope_read_denied**: No test verifies that a scoped viewer gets 404 on detail/export/analytics routes for out-of-scope certs/hosts.
2. **compliance LIKE escape**: No test verifies that a `%` tag parameter doesn't match all rows.
3. **enforce_scope_tag**: No test verifies that a scoped user can't pass an arbitrary tag to compliance reports.
4. **SMTP IP pinning**: No test verifies that the pinned IP is used for connection (only the existing mock-based test covers the SSRF check path).
5. **Renewal digest content**: No test asserts the corrected email body/subject in the global digest or fallback path.
6. **Custom threshold urgent alert**: No test verifies that a 1-day alert fires in digest mode for certs with custom thresholds.
7. **renewal_status skip**: No test verifies that `evaluate_renewal_window` skips certs with `renewal_status='in_progress'` or `'renewed'`.
8. **setup wizard colon validation**: No test verifies the colon rejection in the setup wizard or user management routes.

Each of these is a security or correctness path that a regression test should lock in.

## Problem

The three `test_scan_integration.py` tests (`test_openssl_chain_extraction`, `test_openssl_self_signed_leaf_only`, `test_openssl_connection_refused`) fail because their `_resolve_host` monkeypatch lambdas do not accept the `allowed_subnets` keyword argument that `_scan_host_once` now passes.

Error: `TypeError: <lambda>() got an unexpected keyword argument 'allowed_subnets'`

This is a pre-existing break in the test suite, not a regression in scan logic.

## Fix

The mock signatures need to accept `**kwargs` or include `allowed_subnets` explicitly.

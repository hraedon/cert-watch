# BC-074: Ephemeral signing key invalidates sessions on restart

**Type:** security
**Severity:** medium
**Status:** open
**Filed:** 2026-06-02

## Problem

When `CERT_WATCH_AUTH_SECRET` is not set, `auth.py` generates an ephemeral
`secrets.token_hex(32)` at import time. Every process restart invalidates all
existing sessions. In multi-worker deployments (gunicorn with workers > 1),
each worker has a different key, so sessions created by one worker are invalid
on another.

The same issue applies to `CERT_WATCH_CSRF_SECRET` in middleware.py.

## Suggested fix

Plan 018 B1 (SecurityContext + create_app factory) addresses this by moving
signing key initialization to the lifespan where it can read from a persisted
kv_store. In the interim, log a louder warning at startup when running without
a persistent secret in a multi-worker configuration.

# BC-072: CompositeProvider timing leak on local admin username

**Type:** security
**Severity:** low
**Status:** open
**Filed:** 2026-06-02

## Problem

`_CompositeProvider.authenticate()` tries the local admin provider first. If
the username doesn't match, `LocalAdminProvider` returns immediately with
"invalid credentials" (no scrypt hash computed). If the username does match,
scrypt takes ~100ms. An attacker can distinguish between valid local admin
usernames and invalid ones by timing the response.

## Suggested fix

When the local admin username doesn't match, compute a dummy scrypt hash
against a fixed salt to make timing indistinguishable from the match case.

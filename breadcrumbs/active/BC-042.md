# BC-042: posture.py has zero test coverage and scan_posture table is never populated

**Type:** defect
**Severity:** medium
**Status:** new
**Filed:** 2026-05-30

## Problem

The `posture.py` module (Plan 006 Phase 1) defines `evaluate_posture()` which computes a TLS posture grade (A+/A/B/C/F) from certificate properties. The `scan_posture` table exists in the schema. However:

1. `evaluate_posture()` has **no unit tests** — the module is syntactically valid (ruff passes) but its grading logic (SHA-1 downgrade, RSA key size checks, TLS version penalties) is completely untested.
2. The scan loop (`scan_host()` / `_scan_via_openssl()` in `scan.py`) is **not wired to call `evaluate_posture()`** — the `scan_posture` table is created but never populated. Posture grades never appear in the dashboard or API.

This means the "posture grade" chip on the dashboard and detail page always shows nothing — the feature is structurally present but functionally dead.

## Suggested fix

1. Write `tests/test_posture.py` with tests for each grading rule: TLS 1.0→B, SHA-1→C, RSA<2048→C, self-signed warn, validity>398 days warn, etc.
2. Wire `scan_host()` to call `evaluate_posture()` after extracting the cert, then call `store_scan_posture()` alongside the existing `store_scanned()` call.
3. Verify posture data flows to the dashboard and `/api/posture` endpoint in an integration test.

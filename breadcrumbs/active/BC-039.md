# BC-039: Inline JS in templates growing without automated test coverage

**Type:** todo
**Severity:** low
**Status:** new
**Filed:** 2026-06-01

## Problem

The health banner in `base.html` (~60 lines) and the theme toggle script are inline JavaScript with no unit-test coverage. The only automated guard is the new E2E test (`test_health_banner.py`), which is slower and coarser than a JS unit test. If more widgets follow this pattern, `base.html` will become the same maintenance burden the CSS macros were before they were extracted to `tokens.css`.

## Suggested fix

Extract page-agnostic JS utilities to `static/js/` and add a lightweight JS test runner (Jest or Vitest) for the health banner logic. The banner is simple enough to be a small standalone module.

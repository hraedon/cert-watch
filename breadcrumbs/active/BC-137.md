## Problem

scan.py imports database, alerts, and posture modules, which in turn may import scan.py, creating a circular dependency.

## Resolution

The circular dependency is managed by deliberate inline imports in scan.py:
- `from cert_watch.alerts import WebhookConfig, resolve_pagerduty_for_renewed_cert` (line 687)
- `from cert_watch.database import SqliteTrustAnchorRepository` (line 790)
- `from cert_watch.posture import evaluate_posture` (line 792)

These are import-time-safe because the module-level imports are limited to `certificate_model` and the database init/replace functions, and the runtime inline imports only happen inside the scan and store functions. The design is stable, tested, and documented in AGENTS.md. Resolved by design.

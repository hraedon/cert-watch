# BC-032: Structured JSON logging for observability integration

**Type:** feature
**Severity:** low
**Status:** new
**Filed:** 2026-05-29

## Problem

Logging uses `logging.Formatter` with a plain text format (`%(asctime)s %(levelname)s %(name)s %(message)s`). This works for development but doesn't integrate well with log aggregation systems (ELK, Loki, Datadog, CloudWatch) that expect structured JSON. The cert-watch tool monitors certificates and sends alerts — machine-parseable logs are important for correlating scan failures with alert delivery and scheduler timing.

## Suggested fix

Add a `CERT_WATCH_LOG_FORMAT` env var (`text` default, `json` optional). When set to `json`, use a JSON formatter that includes:

- `timestamp` (ISO 8601)
- `level`
- `logger`
- `message`
- `event` (structured event name, e.g. `scan.completed`, `alert.sent`)
- Any extra fields passed via `logging.LoggerAdapter`

Use `python-json-logger` or a small custom formatter (no new dependency needed for basic JSON output). The `_setup_logging()` function in `app.py` already configures the handler — extend it there.

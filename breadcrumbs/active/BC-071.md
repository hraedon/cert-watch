# BC-071: OAuth userinfo fallback has no nonce binding

**Type:** security
**Severity:** medium
**Status:** open
**Filed:** 2026-06-02

## Problem

When the IdP doesn't return an `id_token`, `complete_oauth_flow` falls back
to the userinfo endpoint to get the username. The userinfo response is obtained
with the access token via bearer auth, but there is no nonce or state binding
— the code only verified that the authorization code exchange was successful,
not that the access token belongs to the current session. An attacker who
obtains any valid access token could use it to authenticate.

## Suggested fix

Log a warning when falling back to userinfo. Consider requiring `nonce` in
the authorization request and verifying it even in the userinfo path, or
document that IdPs must return `id_token` (OIDC-compliant).

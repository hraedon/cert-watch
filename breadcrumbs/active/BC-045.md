# BC-045: OAuth state smuggled in AuthResult.error field

**Type:** quality
**Severity:** low
**Status:** new
**Filed:** 2026-05-30

## Problem

`OAuthProvider.start_oauth_flow()` returns the signed OAuth state in `AuthResult.error` because there is no dedicated field. The route handler (`routes/auth.py:115`) checks `if result.error:` to set the `cw_oauth_state` cookie. This is semantically misleading — `error` always holds a non-empty string on success. A future refactor could easily misinterpret this as a real error and skip setting the cookie, silently breaking OAuth login.

## Suggested fix

Add a dedicated `oauth_state: str = ""` field to `AuthResult`. Update `start_oauth_flow` to use it instead of `error`. Update the route handler to read `result.oauth_state` instead of `result.error`. The `error` field should only be non-empty on actual failures.

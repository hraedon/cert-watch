**Agreed 2026-06-02 (design discussion).** Full RBAC is overkill for cert-watch.
The one distinction worth adding for the target (regulated AD shop) is
**read-only vs read-write**: most staff/auditors can view the cert inventory,
a few can change config and add hosts. Three tiers at most: admin / write / view.

## What already exists (reuse it)

- **Admin tier:** `Settings.admin_users` + `routes/settings._require_admin`
  already gate `/settings` by username. The new write tier should mirror this
  pattern, not invent a new concept.
- **`require_write`** dependency exists (`middleware.py`) — but only `api.py`
  (9 routes) uses it. The form-POST mutations in `routes/hosts.py`
  (`add_host`, `import_hosts`, scan) and `routes/certificates.py`
  (`upload`, delete) gate via middleware-auth + manual `check_csrf`, **not**
  `require_write`. So the mutation chokepoint is partial — those handlers must
  also get the write check (or be routed through `require_write`).

## The wrinkle that sets the cost

The session token carries **only the username** (`{user}:{ts}:{nonce}:{sig}` in
`auth/session.py`) — no groups/roles. Groups are known at login (AuthResult)
but not per request. So:

- **Username-based (cheap, recommended first):** `CERT_WATCH_WRITE_USERS`
  (comma-separated, mirrors `admin_users`). Authenticated users not in the set
  (and not admin) are read-only; mutations return 403. No session-token change.
  Enforce in `require_write` + the form-POST handlers above. Good enough for
  small shops.
- **Group-based (AD-native, more work):** compute the write/admin decision at
  login from `AuthResult.groups` and bake a flag (e.g. `w`/`a`) into the signed
  session token; `require_write` checks the flag. Needed only if write control
  must be driven by AD groups rather than an explicit user list. Touches the
  session format (coordinate with [[BC-080]]/SecurityContext work).

## Scope guard

- Tiers: admin (settings + all), write (default authenticated, or gated by the
  write list/group), view (authenticated, read-only). Nothing finer.
- UI: hide/disable mutation controls for read-only users (cosmetic; the 403 is
  the real gate).
- Independent of [[BC-083]] (auth-on-by-default) — can ship separately.

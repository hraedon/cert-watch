---
identifier: BC-081
title: No server-side session revocation (stateless HMAC, 8h TTL)
kind: security
status: open
severity: medium
---

**ChatGPT review #5.** Sessions are stateless HMAC tokens (`{user}:{ts}:{nonce}:{sig}`)
with an 8h TTL and no server-side store, so a session can't be revoked — rotating
or disabling an LDAP account doesn't invalidate an active cert-watch session for up
to 8h. Real for shops with short credential-rotation policies.

**Fix (deferred, non-trivial):** add a server-side session store (or a per-user
token-version / generation counter checked on each request) so logout / credential
change can invalidate live sessions. Consider shortening the default TTL too.

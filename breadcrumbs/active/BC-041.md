# BC-041: Auth test _reload_app causes module state pollution for subsequent tests

**Type:** defect
**Severity:** low
**Status:** new
**Filed:** 2026-05-30

## Problem

The `_reload_app()` helper in `tests/test_auth.py` uses `importlib.reload()` on `cert_watch.config` and `cert_watch.app`, which replaces module globals like `_signing_key`, `app.state.auth_provider`, and `app.state.settings`. When auth tests that use `_reload_app` (e.g., `test_auth_enabled_redirects_to_login`, `test_authz_gate_denies_user_without_group`, local admin integration tests) run before other test modules like `test_dashboard.py` or `test_rest_api.py`, those modules' tests get 401 responses because the reloaded app still has auth middleware active.

This is the root cause of ~20 test failures when running `pytest tests/` as a whole, even though each module passes individually.

## Suggested fix

`_reload_app` should save and restore module state after each test that uses it, or the auth integration tests should use a separate session/app fixture that doesn't pollute the global module namespace. Alternatives:
1. Use `app.state` restoration (save/restore `auth_provider` and `settings` after each test).
2. Run auth integration tests in a separate process (`pytest-forked` or subprocess isolation).
3. Add an autouse fixture that resets `cert_watch.auth._signing_key` and `cert_watch.app.app.state` after each auth test.

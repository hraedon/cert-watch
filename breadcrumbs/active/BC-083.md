identifier: BC-083
title: Open-by-default auth posture (two permissive defaults stack)
kind: security
status: open
severity: medium
---

**ChatGPT review #1/#3 root.** With no `AUTH_PROVIDER`, all routes are open
(backward-compat, documented, startup warning). Combined with private-scanning-by-
default, an out-of-the-box LAN-exposed instance lets anyone add a host and enumerate
internal TLS certs. The setup wizard is a convenience redirect, not an auth gate, and
is bypassed if any host already exists. This — not the private-IP default — is the
real "any user can add a host" gate.

**Decision needed (user):** whether to harden the default, e.g. require an explicit
`CERT_WATCH_ALLOW_UNAUTH=1` acknowledgement to serve an open instance (vs. refusing /
forcing setup). Flagged for discussion rather than changed unilaterally — it's a
posture/compat trade-off. [[BC-080]] addresses the scan-scope half.

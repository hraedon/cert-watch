## Problem

The install script bootstrapped the app (venv, deps, secrets) but left IIS configuration as manual steps in deploy/iis/README.md.

## Resolution

Added IIS auto-configuration to `scripts/install-windows.ps1` behind a `-ConfigureIIS` switch (opt-in, default off):

1. Checks for WebAdministration module (skips gracefully if missing)
2. Creates site directory and copies/patches `web.config` with actual InstallDir paths
3. Unlocks `system.webServer/handlers` via appcmd (hard failure if it fails)
4. Creates/configures app pool (No Managed Code, AlwaysRunning, no idle/periodic restart)
5. Creates IIS site with HTTPS binding on `*:443:$HostName`
6. Binds TLS cert via `netsh http add sslcert` (idempotent: deletes before add, uses fixed GUID, verifies after)
7. Grants app pool identity read access to site path

New params: `-ConfigureIIS` (switch), `-SitePath`, `-HostName`, `-TlsCertThumbprint`. PS 5.1 compatible (no `??`, ternary, `&&`/`||`, no single quotes in double-quoted strings).

Adversarial review addressed: default-off opt-in (C1), hard failure on unlock (C2), idempotent cert binding with fixed GUID (C3), XML validation + no-BOM write for web.config (C4), accurate footer when IIS not configured (M2).

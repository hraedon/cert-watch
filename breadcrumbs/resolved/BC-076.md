identifier: BC-076
title: Settings._kv_bool let kv_store override an explicit env=0
kind: defect
status: resolved
severity: medium
---

**Resolved 2026-06-02 (ChatGPT review #9).** `_kv_bool` received an already-parsed
bool, so it couldn't tell "env explicitly =0" from "env unset" — a kv_store "1"
silently overrode an operator's explicit `LDAP_START_TLS=0`, violating the
documented "env always wins" contract. Fix: `_kv_bool` now takes the env var name
and checks `env_name in os.environ` first. Tests in `tests/test_allowlist_ssrf.py`.

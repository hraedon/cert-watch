# BC-029: REST API pagination lacks HATEOAS navigation links

**Type:** design-question
**Severity:** low
**Status:** resolved
**Filed:** 2026-05-28
**Resolved:** 2026-05-29

## Resolution

Added HATEOAS pagination links (`self`, `next`, `prev`) to all paginated REST API responses (`/api/certificates`, `/api/hosts`, `/api/alerts`). Links are derived from the request base URL. This is a non-breaking additive change — existing `page`, `limit`, `total`, `pages` fields are preserved. `next` is `null` on the last page, `prev` is `null` on the first page.

**Resolved:** 2026-05-30

## Problem

`git diff` shows `database.py` is effectively empty in the working tree (was ~1,136 lines). The replacement modules (`database/schema.py`, `database/repo.py`, `database/queries.py`) already exist and appear complete, suggesting this is unfinished work toward BC-035 (database.py monolith decomposition).

Because it is uncommitted, there is a risk that:
- The deletion is accidental and will be lost on the next checkout.
- A future agent or human contributor will see the empty file and assume the project is broken.
- The `__init__.py` re-export layer may not be wired correctly (imports in other modules still reference `from cert_watch.database import X`).

## Suggested fix

Before the next release or significant change, verify whether the decomposition is complete and functional. If yes: commit the deletion of `database.py` alongside any import-path updates. If no: revert the working-tree deletion so the repo stays in a consistent state until BC-035 is finished.

## Resolution (2026-05-30)

The decomposition was already committed in a prior session. The working tree is clean — `database.py` was replaced by the `database/` package with `schema.py`, `repo.py`, `queries.py`, `connection.py` and `__init__.py` re-exporting all public names. This session further refactored `schema.py` into `ensure_base()` + `init_schema()` with the migration runner. No action needed.

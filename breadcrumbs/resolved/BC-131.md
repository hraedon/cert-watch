## Problem

The certificate detail page does not have an on-demand "Check revocation" button that triggers OCSP/CRL checks interactively, as shown in the 2026-06-02 design.

## Resolution

Already implemented. The certificate detail page (`certificate_detail.html`) includes:
- A "Check revocation" button with `data-action="check-revocation"`
- A `checkRevocation()` JavaScript function that calls `/api/certificates/{cert_id}/revocation`
- Global event delegation in `base.html` that routes the button click to the handler
- The API endpoint `GET /api/certificates/{cert_id}/revocation` in `routes/api/certificates.py`
- Tests covering the button presence (`test_certificate_detail_revocation_button`) and the API endpoint (`test_api_revocation_check`).

This breadcrumb was filed before the feature was added; the status has been corrected to resolved.

**Resolved 2026-06-02 (ChatGPT review #1, revised).** Added
`CERT_WATCH_ALLOWED_SUBNETS` — a CIDR allowlist scoping which private ranges are
scannable. When set, a private target is allowed only if inside the list; public
stays scannable, loopback/link-local/metadata stay blocked. Non-breaking (tightens
only when set). Enforced in `_is_blocked_ip` and threaded through scan.py +
`_is_blocked_host_check`; setup wizard prompts for ranges; merged from kv_store.
Deliberately did NOT flip the global `allow_private` default to deny — that breaks
the primary internal-monitoring use case on upgrade for a low payoff (the scan
reads cert metadata, not response bodies/creds). See [[BC-083]] for the related
open-by-default *auth* posture, which is the actual "any user can add a host" gate.

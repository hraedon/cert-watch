## Problem

Test helpers for seeding fleets used raw SQL instead of the repository or upload helpers, making tests more fragile.

## Resolution

Created `tests/_helpers.py` with three repository-backed helpers:
- `seed_certificate(db_path, cert, **kwargs)` — wraps `SqliteCertificateRepository.add()` with optional deterministic `cert_id` and `tags`
- `seed_scanned(db_path, hostname, port, leaf, chain=[])` — wraps `store_scanned(ScannedEntry(...), db_path)`
- `seed_host(db_path, hostname, port)` — wraps `SqliteHostRepository.add()`

Refactored four test files to use the helpers instead of raw SQL:
- `test_renewal_window.py` — `_insert_cert()` now uses `seed_certificate()` with deterministic `cert_id`
- `test_alerts.py` — `_insert_cert()` now uses `seed_certificate()`
- `test_compliance.py` — `_seed_fleet()` now uses `seed_certificate()` + `store_scan_posture()`
- `test_drift.py` — `_seed_history()` now uses `record_cert_history()` instead of raw INSERT into `cert_history`

Also added `tests/__init__.py` so `tests._helpers` is importable.

## Related

- `test_dashboard_queries.py` already used proper helpers (`store_scanned`, `SqliteCertificateRepository`) — it was the model for the refactor.

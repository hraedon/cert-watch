## Problem

The Discover page (`routes/views.py`) used hardcoded issuer name fragments (`NOT LIKE '%Let%'`, `NOT LIKE '%ISRG%'`, etc.) to identify private-CA hosts. This was brittle and would miss private CAs with unexpected names.

## Resolution

Replaced hardcoded issuer fragments with trust-anchor-based detection. The `scan_posture.chain_status` field now stores `'private'` when a leaf cert chains to a known trust anchor (via `cert_chain.py` chain validation). The Discover page counts private-CA hosts by querying `chain_status = 'private'` in `scan_posture` (migration 0016). This is accurate for any issuer name as long as the trust anchor table is populated.

## Problem

Plan 040's roles/users foundation (migration 0019, `users_roles.py`, RBAC system) was merged, but the team dashboard and alert routing were still outstanding.

## Resolution

Implemented team dashboard and role-based alert routing:

- **Team Dashboard** (`GET /team`): Authenticated users see certificates owned by their role's team email. Includes chain-aware urgency computation, SQL-level pagination (25 per page), and stat cards matching the main dashboard. Shows empty state for users with no role or no matching certs. Pre-migration databases gracefully handled.
- **Role-based Alert Routing** in `evaluate_all_certs()`: When a role's email matches a host's owner_email, all users in that role receive alert notifications via their user emails. Single-pass query (no N+1), graceful fallback when tables don't exist.
- **Nav link** in base.html: "Team" link visible to authenticated users.

Files changed: `routes/views.py`, `alerts.py`, `middleware.py`, `templates/team_dashboard.html`, `templates/base.html`, `tests/test_no_inline_styles.py`.

Tests: 14 tests in `tests/test_team_dashboard.py` covering happy path, pagination, urgency, missing tables, alert routing, and nav visibility.

Adversarial review addressed: pagination (H1), chain-aware urgency (H2), stat card labels (H3), logging on failure (H5), pre-migration error handling (H6), N+1 query (H7).

---
identifier: BC-060
title: OAuth error messages leak raw exception details to users via URL
kind: security
status: resolved
severity: medium
---

## Problem

Raw exception messages from JWT verification and token exchange were included in `AuthResult.error`, which flowed into `/login?error=...` URL parameters visible in browser history and proxy logs.

## Resolution

Replaced specific exception details in error messages with generic strings:
- `ID token verification error: {exc}` → `ID token verification failed`
- `OAuth authentication failed: {exc}` → `OAuth authentication failed`

Detailed exceptions are still logged server-side via `logger.warning()`.

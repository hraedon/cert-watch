## Problem

`build_auth_provider` silently fell back to `NoAuthProvider` (open access)
when LDAP or OAuth was configured incorrectly (e.g., missing LDAP_SERVER or
OAUTH_CLIENT_ID). An operator who set AUTH_PROVIDER=ldap but forgot LDAP_SERVER
would get zero authentication instead of a hard failure. Unknown provider
values also fell back to NoAuthProvider.

## Resolution

`build_auth_provider` now raises `ValueError` for misconfigured or unknown
providers instead of silently degrading. Tests updated to expect ValueError.

**Resolved 2026-06-03.** Added `tests/test_oauth_callback.py` — 10 unit tests
driving every branch of `oauth_callback` at the `AuthProvider` boundary: no
provider, provider `error=`, missing code, missing state cookie, bad-signature
state cookie, state/cookie mismatch, token-exchange failure, authz denial vs.
admission (`CERT_WATCH_ALLOWED_GROUPS`), and the happy path. Each failure case
asserts **no session cookie is minted**; the happy path asserts the session is
issued and the one-time state cookie cleared. `routes/auth.py` coverage rose
52% → 83%, with the callback block (lines 170-252) now fully covered. The
remaining uncovered lines are `oauth_start` and the logout session-bump, not the
callback.

## Problem

`routes/auth.py` sits at ~52% coverage, and the largest uncovered block is the
OAuth callback handler `oauth_callback` (`routes/auth.py:170-252`) — the code
that validates the signed `state`/`cw_oauth_state` cookie, exchanges the
authorization `code`, verifies the ID token, and mints the session. This is the
single most security-sensitive request path in the app (a bug here is an auth
bypass or session-fixation vector), yet it is currently exercised only
indirectly via E2E flows, not by unit tests with adversarial inputs.

`test_auth.py` covers `_sign_state` and `AuthResult` shaping but does not drive
the callback handler itself. Coverage being lowest on the most dangerous code
means the README's "hardened" framing is slightly ahead of the test evidence
here.

## Suggested fix

Add unit tests for `oauth_callback` covering at least:

- **state forgery / tampering** — a `state` param not matching the signed
  `cw_oauth_state` cookie is rejected (no session issued).
- **missing state cookie** — callback with `state` but no cookie is rejected.
- **provider `error=` passthrough** — redirects to `/login?error=...` and
  clears the state cookie.
- **happy path** — valid code+state mints a session and sets the session
  cookie (mock the provider's token/userinfo exchange).
- **token-exchange failure** — provider raising mid-exchange yields a clean
  login redirect, not a 500 or a partial session.
- **authorization gates** — a user outside `CERT_WATCH_ALLOWED_GROUPS` /
  `CERT_WATCH_ALLOWED_ROLES` is denied after a successful authentication.

Mock the OAuth provider at the `auth_provider` boundary (as the E2E conftest
already does) so these stay fast unit tests. Target: lift `routes/auth.py`
coverage of the callback block from ~0% to near-complete and make the
state-validation assertions explicit.

identifier: BC-067
title: OAuth state bypassed when empty string; open redirect via Host header
kind: security
status: resolved

## Problems fixed

1. **OAuth state bypass when empty**: `complete_oauth_flow` had `if state:`
   which skipped state verification when state was empty string. Changed to
   `if not state:` → immediate rejection.

2. **Open redirect via Host header**: OAuth `redirect_uri` was built from
   `request.base_url` which derives from the `Host` header. Behind a proxy
   with CERT_WATCH_TRUST_PROXY=1, an attacker-controlled X-Forwarded-Host
   could redirect the OAuth callback to an attacker server. Added
   `CERT_WATCH_BASE_URL` env var; `_get_base_url()` prefers it over
   `request.base_url`.

3. **CSRF cookie/session cookie missing path=/**: Added `path="/"` to all
   `set_cookie` and `delete_cookie` calls for `cw_auth`, `cw_sid`, and
   `cw_oauth_state`.

4. **Break-glass detection used `type().__name__`**: Changed to `isinstance()`
   checks to prevent silent breakage on rename/refactor.

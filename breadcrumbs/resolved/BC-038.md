## Problem

Each template (dashboard.html, alerts.html, scan_history.html, certificate_detail.html, login.html) repeats the same ~30-line `svg_icon` macro definition. This is ~150 lines of noise and makes adding a new icon a 5-file edit. It also increases the risk of drift if one copy is updated and others are missed.

## Suggested fix

Extract the macro into `src/cert_watch/templates/macros/icons.html` and import it:

```jinja
{% import "macros/icons.html" as icons %}
{{ icons.svg_icon('shield', 16) }}
```

All existing templates can be updated in one pass; no functional change.

## Resolution

Resolved in session 2026-05-30. Extracted the `svg_icon` macro to `src/cert_watch/templates/macros/icons.html` (superset of all icons across templates: shield, clock, alert, check-circle, search, plus, upload, refresh, server, file, globe, chevron-right, download, filter, link, trash, x, chevron-down, key, check, bell). Updated all 4 templates (dashboard, certificate_detail, alerts, scan_history) to use `{% import "macros/icons.html" as icons %}` and `{{ icons.svg_icon(...) }}` instead of inline macro + `{{ svg_icon(...) }}`. Login.html didn't use the macro (it has one inline SVG). All 330 tests pass, lint clean.

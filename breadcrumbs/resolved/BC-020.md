## Problem

Hosts that are pending or have never been scanned lack a place for operators to attach notes about why they are pending, expected resolution timeline, or owner context.

## Resolution

Implemented in 2026-06-08 session:
- Added `notes` column to `hosts` table (schema + ALTER TABLE migration for existing DBs)
- Updated `HostEntry` dataclass and `SqliteHostRepository` to support `notes` (add, get, list, update)
- Added `POST /hosts/{host_id}/notes` form route and `PATCH /api/hosts/{host_id}/notes` API endpoint
- Updated `add_host` and `import_hosts` routes to accept notes (form field + CSV column)
- Exposed host notes in the certificate detail page (`certificate_detail.html`) with inline edit/save toggle
- Added host notes indicator in dashboard expandable host rows and normal entries
- Updated hosts CSV export to include the `notes` column
- Added 16 tests covering repository roundtrip, API endpoints, form routes, CSV export, and UI rendering
- All 1371 tests pass; ruff clean.

## Problem

The `_rate_db_initialized` guard was added to `check_rate_limit()` in `middleware.py` to prevent `init_schema` from being called on every rate-limited request. There is currently no test that verifies `init_schema` is only invoked once.

## Suggested fix

A simple test that calls `check_rate_limit()` twice and asserts `init_schema` was only called once would prevent regression.

This is a test gap, not a code defect.

**Resolved 2026-06-02 (ChatGPT review #8).** The custom-DNS path
(`CERT_WATCH_DNS_SERVERS`) checked txid + rcode but skipped the question section
without verifying it. A spoofed response with a matching txid could be accepted.
Now the first question must echo the queried name + qtype. Mitigated already by
pinned-IP + random txid; this closes the blind-spoof gap on the opt-in DNS path.

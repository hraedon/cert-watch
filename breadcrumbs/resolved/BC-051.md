# BC-051: New `oauth_state` field on `AuthResult` may break existing test mocks

**Type:** quality
**Severity:** low
**Status:** resolved
**Filed:** 2026-05-30
**Resolved:** 2026-05-30

## Problem

BC-045 added a new field `oauth_state: str = ""` to `dataclass AuthResult`. Dataclass behavior means `__init__` accepts this parameter, and `repr` / `eq` include it. Any code constructing `AuthResult` with positional args beyond `username` could shift indices. Mocks inspecting `result.error` after OAuth login needed updating.

## Resolution

1. Added regression tests in `tests/test_auth.py`:
   - `test_authresult_oauth_state_explicit`: `AuthResult(success=True, redirect_url="...", oauth_state="signed").oauth_state == "signed"`
   - `test_authresult_oauth_state_default`: `AuthResult(success=True, error="something").oauth_state == ""`

2. Updated AGENTS.md `Architecture notes` → `auth.py` section to document `oauth_state` as a field on `AuthResult` alongside `groups` and `roles`.

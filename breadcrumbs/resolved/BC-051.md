# BC-051: A+ posture grade unreachable from scans (HSTS detection missing)

**Type:** defect
**Severity:** low
**Status:** resolved
**Filed:** 2026-05-31

**Resolved:** 2026-05-31 — scan_host() now probes HSTS via HTTP HEAD request on port 443 after TLS handshake. _probe_hsts() checks Strict-Transport-Security header. A+ posture grade now reachable.

## Problem

`posture.py:220-223` documents that A+ requires TLS 1.3 + HSTS, but scans never probe HTTP response headers for HSTS. The `hsts` parameter is only ever non-None when a caller passes it explicitly (e.g. detail-page fallback). No scan path ever sets it, so A+ is unreachable from scan-stored posture data.

This means the highest grade any scanned cert can receive is A, even with TLS 1.3 and perfect cert properties.

## Suggested fix

Extend `_scan_host_once()` or `_scan_via_openssl()` to make an HTTP request after the TLS handshake (when port 443) and check for the `Strict-Transport-Security` response header. Set `hsts=True` on the posture result when present.

This is Plan 006 Phase 1 scope — the comment in posture.py explicitly defers it. Low priority since A vs A+ is cosmetic.

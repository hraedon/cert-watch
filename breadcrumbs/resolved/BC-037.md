# BC-037: Dashboard template has 400 lines of inline styles, will be unmaintainable with grouping

**Type:** design-debt
**Severity:** low
**Status:** resolved
**Resolved:** 2026-05-30
**Filed:** 2026-05-29

## Problem

`dashboard.html` is ~400 lines with inline `style=""` attributes on nearly every element. No CSS classes for layout, spacing, or component styling — just ad-hoc inline declarations. As BC-033 (fingerprint grouping with collapsed/expanded rows) lands, the template complexity will grow significantly. Toggle state, per-host status indicators, and group headers all need consistent styling. With inline styles, this becomes a maintenance burden and a source of inconsistencies.

## Suggested fix

Extract inline styles into a CSS class system in the existing `static/` stylesheet. Use BEM or utility-class conventions already implied by the `cw-` prefix (e.g., `cw-row`, `cw-chip`, `cw-pill`). This is a prerequisite for BC-033 to land cleanly.

## Resolution

Prior sessions had already extracted most inline styles into `static/css/tokens.css` (844 lines of design tokens, utility classes, and component styles with `cw-` prefix). This session removed the remaining 8 static inline styles from templates:
- `audit.html`: Replaced `style="display:flex;gap:12px;..."` with utility classes, `style="width:auto"` with `.cw-w-auto`, `style="width:140px"` with `.cw-w-140`
- `certificate_detail.html`: Replaced 5x `style="min-width:16px"` with `.cw-min-w-16`, `style="margin-left:auto"` with `.cw-ml-auto`

All remaining 13 inline styles are dynamic CSS custom property bindings (`style="--tone:{{ _c }}"`) — the correct pattern for Jinja2 → CSS value injection.

kind: bug
status: resolved
severity: low
---

**Found 2026-06-02 code audit.** The `GET /api/export/certificates.csv` endpoint
included `leaf_subject`, `leaf_issuer`, and `leaf_not_after` columns that simply
repeated the row's own `subject`, `issuer`, and `not_after` values. For chain
rows, the "leaf" columns contained the *parent* cert's data (not the actual leaf),
making the export misleading for compliance reporting.

**Fix:** Removed the duplicate `leaf_*` columns. The CSV now has a clean schema:
host, source, subject, issuer, not_after, days_remaining, urgency, chain_valid.
Chain rows are still included as separate rows with the same host. All 694 tests pass.

# BC-032: Structured JSON logging for observability integration

**Type:** feature
**Severity:** low
**Status:** resolved
**Filed:** 2026-05-29
**Resolved:** 2026-05-29

## Resolution

Added `CERT_WATCH_LOG_FORMAT` env var (`text` default, `json` optional). When set to `json`, logs are emitted as structured JSON with `timestamp`, `level`, `logger`, `message`, and `extra` fields. The `_setup_logging()` function in `app.py` now accepts a `log_format` parameter and uses a custom `_JsonFormatter` class that serializes log records to JSON. No new dependencies were added. Non-breaking: default remains text format.

# BC-065: DNS parser vulnerable to infinite loop on crafted compression pointer cycles

**Type:** defect
**Severity:** medium
**Status:** open
**Filed:** 2026-06-01

## Problem

`_parse_dns_name` in scan.py follows DNS compression pointers without
loop detection. A malicious DNS response with circular pointers
(e.g., offset A → offset B → offset A) causes an infinite loop,
blocking the scan thread. Reachable when CERT_WATCH_DNS_SERVERS is
configured and the resolver contacts a compromised DNS server.

## Fix applied (2026-06-01)

Added `seen_offsets` set to `_parse_dns_name` to detect and break on
pointer cycles. Also added DNS response TXID validation (match query
ID) and RCODE check (reject non-zero) in `_resolve_with_dns`.

---
identifier: BC-084
title: A+ posture grade unreachable for non-443 TLS endpoints
kind: design
status: resolved
severity: low
---

**ChatGPT review #11.** `_probe_hsts` returns None for `port != 443`, and the
posture grader requires `hsts is True` for A+ — so a genuinely well-configured
internal TLS service on a non-443 port can never reach A+. Defensible (HSTS is a
443/HTTP concept) but the grade is then slightly misleading for non-443.

**Fix (low):** either document A+ as 443-only in the posture output, or decouple the
A+ criterion from HSTS for non-443 ports.

**Resolved 2026-06-02.** Added `port: int = 443` parameter to `evaluate_posture()`
(`posture.py`).  A+ now requires TLS 1.3 + (HSTS present OR port != 443).  This
makes the grade accurate for non-443 TLS services while keeping the stricter
HSTS requirement for standard HTTPS endpoints.  Callers updated: `scan.py`
passes `entry.port` to `evaluate_posture()`.  New tests added:
`test_a_plus_grade_for_tls13_non_443_no_hsts` and
`test_a_grade_for_tls13_443_no_hsts`.

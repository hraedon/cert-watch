# BC-050: Missing database indexes for scan_history and alerts pagination

**Type:** quality
**Severity:** low
**Status:** resolved
**Filed:** 2026-05-30
**Resolved:** 2026-05-30

## Problem

BC-044 added SQL-level pagination to `list_scan_history` and `list_alerts_with_subject`, but neither `scan_history` nor `alerts` had an index on the columns used for sorting.

## Resolution

Added three indexes to `_BASE_INDEXES` in `database/schema.py`:

```sql
CREATE INDEX IF NOT EXISTS idx_scan_history_scanned_at ON scan_history(scanned_at DESC);
CREATE INDEX IF NOT EXISTS idx_alerts_created_at ON alerts(created_at DESC);
CREATE INDEX IF NOT EXISTS idx_alerts_status_created ON alerts(status, created_at DESC);
```

Also added regression test `test_init_schema_creates_scan_history_and_alerts_indexes` in `tests/test_database.py`.

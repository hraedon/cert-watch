**Resolved:** 2026-05-31 — /metrics now uses targeted SQL (SELECT hostname, port, subject, not_after) instead of list_dashboard_rows(). COUNT(*) for host/cert/expired counts.

## Problem

The `/metrics` endpoint (`views.py:322`) calls `list_dashboard_rows(db)` which materializes every certificate row, and `SqliteHostRepository(db).list_all()` which loads all hosts. For a large fleet this is the same O(n) memory concern as BC-047/BC-048.

`/metrics` is typically scraped every 15-30s by Prometheus, so this runs far more frequently than the dashboard and amplifies memory pressure.

## Plan

Resolved by Plan 012 §2.3.

## Suggested fix

Replace `list_dashboard_rows()` with targeted SQL queries:
- `cert_watch_cert_expiry_days` — iterate a cursor over `SELECT host, subject, days_remaining ...` instead of materializing a list.
- `cert_watch_hosts_tracked` — `SELECT COUNT(*) FROM hosts`.
- `cert_watch_certificates_tracked` — `SELECT COUNT(*) FROM certificates WHERE is_leaf = 1`.
- `cert_watch_certificates_expired` — `SELECT COUNT(*) FROM certificates WHERE is_leaf = 1 AND julianday(not_after) <= julianday('now')`.

This avoids building any list in Python.

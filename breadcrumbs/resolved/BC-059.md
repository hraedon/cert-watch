# BC-059: LDAP STARTTLS without CA cert silently falls back to CERT_NONE

**Type:** security
**Severity:** medium
**Status:** resolved (2026-05-31)
**Filed:** 2026-05-31

## Problem

In `auth.py:225`, when `LDAP_SERVER` uses `ldap://` with `START_TLS=True` but no `LDAP_CA_CERT` is configured, the TLS context falls back to `ssl.CERT_NONE`. This means the STARTTLS upgrade does not verify the server certificate, enabling MITM attacks on the LDAP bind credentials.

## Resolution

Changed `_build_tls()` to always use `ssl.CERT_REQUIRED` for both `ldaps://` and STARTTLS paths. When STARTTLS is used without `LDAP_CA_CERT`, the system trust store is used (same as LDAPS without CA cert). Added a warning log for STARTTLS without CA cert reminding operators to set `LDAP_CA_CERT` for private CAs.

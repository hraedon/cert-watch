## Problem

The current `/healthz` endpoint does everything: DB connectivity, scheduler thread health, certificate counts, last scan status. The `deployment.yaml` has both `livenessProbe` and `readinessProbe` hitting `/healthz`.

Kubernetes best practice is:
- `/healthz` (liveness) = "process is alive" (lightweight)
- `/readyz` (readiness) = "DB is reachable and scheduler has run at least once" (heavier)

The current setup means the liveness probe does expensive DB queries on every 30-second cycle.

## Suggested fix

Split `/healthz` into `/healthz` (lightweight) and `/readyz` (full), and update `deployment.yaml` probes accordingly.

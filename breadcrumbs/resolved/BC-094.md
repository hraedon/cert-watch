## Problem

`queries.py` was 2,462 lines, bundling dashboard queries, aggregate helpers, encryption utilities, kv_store operations, drift detection, posture queries, pagination, fleet pivot, calendar, cert_history, and session versions into a single monolithic file.

## Resolution

Decomposed into 10 concern-specific modules under `src/cert_watch/database/`:

1. `encryption.py` — at-rest encryption (derive_encryption_key, fernet_encrypt, fernet_decrypt, check_encrypted_values, re_encrypt_kv_store)
2. `cert_ops.py` — certificate store operations (replace_scanned, delete_certificate_cascade, get_renewal_history, _compute_renewal_diff, distinct_tags)
3. `drift.py` — drift detection + cert_history (DriftEvent, detect_drift, create_drift_alert, record_cert_history, purge_old_history, list_cert_history, list_tls_version_trends, list_grade_trends)
4. `dashboard.py` — dashboard + unified entry queries (list_dashboard_rows, list_dashboard_page, list_dashboard_grouped_page, get_cert_detail, _build_unified_from_dash, _load_unified_filtered, etc.)
5. `posture.py` — posture evaluation storage/retrieval (store_scan_posture, get_posture_for_cert, get_posture_grades_for_certs, get_posture_for_certs)
6. `kv_store.py` — kv_store helpers (kv_get, kv_set, kv_set_secret, kv_all)
7. `session_versions.py` — session token revocation (get_session_version, bump_session_version)
8. `pagination.py` — alert/scan history pagination (list_alerts_with_subject, list_scan_history, _total_alerts, _total_scan_history, purge_old_alerts)
9. `fleet.py` — fleet pivot + grouping (list_fleet_pivot, get_pivot_group_entries, group_entries_by_fingerprint)
10. `calendar.py` — calendar view queries (list_calendar)

`queries.py` remains a thin re-export layer for backward compatibility. `database/__init__.py` now imports from the submodules directly. All 1218 tests pass with 88.83% coverage.

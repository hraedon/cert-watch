**ChatGPT review #10.** `_has_native_chain_api()` only returns True on Python 3.13+;
the Dockerfile ships `python:3.12-slim`, so the production image always takes the
`openssl s_client` fallback and the native-chain path has no real-world coverage there.
Not a bug — a coverage gap.

**Options:** bump the container to 3.13 (gains native path, drops the openssl dep on
the hot path), or add a CI matrix leg on 3.13 to exercise the native path, or document
that native extraction is dev-only until the base image moves to 3.13.

**Resolved 2026-06-02.** Bumped both Dockerfile stages (`builder` and `runtime`) from
`python:3.12-slim` to `python:3.13-slim`.  Updated CI workflow (`ci.yml`) to
`python-version: "3.13"`.  Updated ruff `target-version` to `py313` in
`pyproject.toml`.  The native chain-extraction path (`SSLSocket.get_unverified_chain`)
is now active in the production image, removing the openssl binary dependency on the
hot path.  `requires-python` remains `>=3.12` for backward compatibility with existing
installs.

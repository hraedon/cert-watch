kind: bug
status: resolved
severity: medium
---

**Found 2026-06-02 code audit.** In `routes/api.py`, the `POST /api/alert-groups`
endpoint called `_validate_webhook_url(webhook_url)` but discarded its return
value. When the validation returned a `JSONResponse` error (invalid scheme,
private IP, etc.), the code continued and saved the invalid URL to the database.

The `PATCH /api/alert-groups/{group_id}` endpoint correctly checks the return
(`if err: return err`), so this was an inconsistency in the create path only.

**Fix:** Added `if err: return err` after the validation call. All 694 tests pass.

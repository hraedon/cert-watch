# BC-062: HSTS probe re-resolves hostname with no SSRF blocklist (rebinding/SSRF bypass)

**Type:** defect
**Severity:** medium
**Status:** resolved (2026-06-01)
**Filed:** 2026-05-31

## Problem

`_probe_hsts` opened its own HTTPS connection by hostname, bypassing the
pinned_ip from the SSRF check. DNS rebinding attack vector.

## Resolution

`_probe_hsts` now accepts `pinned_ip` parameter. When provided, connects to
the pinned IP and sets `Host` header to the original hostname. All call sites
in `_scan_host_once`, `_scan_host_via_openssl`, and the Python fallback now pass
`pinned_ip` through. This aligns the HSTS probe with the same DNS rebinding
protection used by the TLS scan path (BC-053).

The secondary issue (HSTS probe uses verify=True, making it useless for
self-signed/internal hosts) is noted but not yet addressed.

**Resolved 2026-06-03.** Added `cert_watch/firstrun.py` — a leaf module with two
pure functions: `is_network_exposed(bind_host, trust_proxy)` and
`first_run_action(*, has_provider, allow_unauth, network_exposed)` returning a
`FirstRunPosture` enum (`AUTHENTICATED` / `SERVE_OPEN` / `PROVISION_ADMIN`).
`app.lifespan` now computes the posture via these and only performs the side
effects; the fail-closed case is "still `PROVISION_ADMIN` after the provisioning
attempt," which preserves prior behavior exactly (including the injected-NoAuth
escape hatch). New `tests/test_firstrun_posture.py` table-tests all 8
`(has_provider, allow_unauth, network_exposed)` combinations plus exposure
detection; `test_bc083_081.py` continues to cover the end-to-end lifespan path
(actual provisioning + fail-closed SystemExit).

## Problem

The most security-critical decision in the app — whether a fresh, no-auth
instance serves open, auto-provisions an admin, or refuses to start — is
computed inline inside `app.py:lifespan()` (the `network_exposed` derivation at
`app.py:185-187` and the two gates at `app.py:194-199` and `app.py:227-239`).
The decision is a pure function of a handful of inputs (bind host, trust_proxy,
allow_unauth, whether a provider resolved, whether provisioning persisted), but
it is entangled with side effects (schema init, logging, scheduler start), so it
can only be tested through the heavyweight lifespan path.

This is **not a coverage gap** — `test_bc083_081.py` already exercises the key
combinations (nonloopback+noauth → provision, loopback+TRUST_PROXY → provision,
bare loopback → open+setup, allow_unauth → open, provisioning-fail → fail
closed, auth-configured → start). It is a maintainability/clarity issue: the
branching for the single most important security decision lives in a
side-effecting async context manager, and BC-090 was already one bug where the
`--host`/env inputs to this decision could diverge from the real bind.

## Suggested fix

Extract a pure helper, e.g.:

```python
class FirstRunPosture(enum.Enum):
    SERVE_OPEN = "serve_open"        # bare-loopback dev / allow_unauth
    PROVISION_ADMIN = "provision"    # network-exposed, no provider
    FAIL_CLOSED = "fail_closed"      # network-exposed, provisioning failed

def decide_first_run_posture(
    *, bind_host: str, trust_proxy: bool, allow_unauth: bool,
    has_provider: bool, provisioned_ok: bool,
) -> FirstRunPosture: ...
```

Then `lifespan()` calls it and only performs the side effects. Add one
table-driven test enumerating every input combination (the full truth table),
so the decision is verified independently of schema/scheduler setup. Keep the
existing `test_bc083_081.py` integration tests as the end-to-end check.

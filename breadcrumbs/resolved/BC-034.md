# BC-034: Owner/contact field for certificates and hosts with alert routing

**Type:** feature
**Severity:** medium
**Status:** resolved
**Filed:** 2026-05-29
**Resolved:** 2026-05-29

## Resolution

Added structured owner/contact fields to the `hosts` table and related UI/API surfaces:

### Schema
- Added `owner_name`, `owner_email`, `owner_slack`, `renewal_status` columns to the `hosts` table with backward-compatible migrations.
- `HostEntry` dataclass updated with new fields.

### API
- `GET /api/hosts` now includes `owner_name`, `owner_email`, `owner_slack`, `renewal_status` in the response.
- `PATCH /api/hosts/{host_id}/owner` — new endpoint to update owner info and renewal status.
- `GET /api/export/hosts.csv` now includes owner fields.
- HATEOAS links added to `/api/hosts` (BC-029).

### Alerts
- Alert messages now include owner name, email, and Slack when set.
- `extra_recipients` field on Alert dataclass routes alert emails to owner in addition to global recipients.
- Webhook payload includes `extra_recipients` field.
- `renewal_status = "renewed"` suppresses alert creation entirely.
- `renewal_status = "in_progress"` marks alert as "renewal in progress" in the message.

### Dashboard
- Owner name shown as a chip next to the host name.
- Renewal status shown as colored chips ("renewing" in yellow, "renewed" in green).

identifier: BC-118
title: AD-only group OID hardcoded — transitive group checks break on OpenLDAP/FreeIPA
kind: design-debt
status: resolved
severity: medium
---

## Problem

`auth/ldap_provider.py` builds the required-groups filter with the hardcoded AD matching-rule OID `1.2.840.113556.1.4.1941` (AD transitive membership). This OID is AD-specific. On OpenLDAP or FreeIPA it is unsupported, so `LDAP_REQUIRED_GROUPS` checks silently fail.

## Resolution

Configurable group filter is fully implemented:
- `LDAP_GROUP_FILTER` env var (default empty) parsed by Settings and threaded through auth factory.
- `auth/ldap_provider.py` `_build_group_filter` uses `self.group_filter` as a template with `{group}` placeholder.
- Default behavior: when `group_filter` is empty, the AD transitive OID is used for backward compatibility.
- Custom: `LDAP_GROUP_FILTER=memberOf={group}` for plain OpenLDAP memberOf checks, or `member={group}` for posixGroup.
- Tests: `test_bc118_default_uses_ad_transitive_oid` and `test_bc118_custom_group_filter` cover both paths.

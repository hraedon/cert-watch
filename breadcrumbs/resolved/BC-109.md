## Problem

The current `/metrics` endpoint only emits success-state gauges (`cert_watch_cert_expiry_days`, `hosts_tracked`, `certificates_tracked`, `expired`). There is no counter for scan failures.

## Suggested fix

Add `cert_scan_errors_total{host="...",reason="connection_refused|timeout|dns_failure|blocked"}` counter to make the Prometheus alert rules far more useful. Currently operators cannot alert on "scanning is broken" without inspecting logs.

The endpoint is hand-rolled OpenMetrics text; adding a counter requires extending the metrics string builder in `routes/views.py`.

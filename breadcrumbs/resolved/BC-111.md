## Problem

The `/metrics` endpoint is hand-rolled OpenMetrics text (lines.append with `# HELP`, `# TYPE`, and gauge values). This is fragile for label escaping, and makes it hard to add `Histogram`, `Summary`, or `Counter` types.

## Suggested fix

Using the `prometheus_client` library would provide:
- Correct label escaping
- Support for `Histogram` for scan duration distribution
- Trivially extendable endpoint

The test suite already validates the output format (`test_metrics.py`). The trade-off is a new dependency (`prometheus_client` is ~3MB, pure Python).

This is a long-tail improvement, not a defect.

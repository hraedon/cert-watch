## Problem

The `certificates.py` drift loop reuses variable name `c` as both `Certificate` and `dict`, causing mypy index/assignment errors.

## Resolution

The certificates.py drift loop no longer reuses a single variable name. The current code uses:
- `cr` for chain database rows, `c` for `_row_to_cert(cr)` results (consistent `Certificate` type)
- `curr` / `prev` for history entry dicts in the drift comparison loop
- `mypy src/cert_watch/routes/certificates.py` reports zero errors.

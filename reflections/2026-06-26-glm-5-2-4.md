## On the work done

The compounding win was WI-092 + the bonus `_store_sem` removal. WI-092 (acquire `get_write_lock` in `store_scanned_async`'s worker thread) made the module-level `asyncio.Semaphore` in `hosts.py` redundant — and that semaphore was a latent cross-event-loop bug (`RuntimeError: bound to a different event loop`) that was failing a test on clean HEAD. One WI's hardening obsoleted a footgun. I'm confident in the deadlock analysis: the lock lives only in the async wrapper (not in `store_scanned`), so the scheduler's already-locked direct call can't re-enter a non-reentrant `threading.Lock`.

I'm most wanting a second pair of eyes on WI-116. My reachability claim is that the legacy `admin_users` fail-open is *unreachable* today because `build_auth_context` returns `AuthContext.full_access` (is_admin=True) for the no-role-map case, so `ctx.is_admin` grants admin before the legacy branch. I traced the session + form paths and the API-key path (handled separately in `_write_denied`). The fix is behavior-preserving if that trace is right, and the regression test guards the invariant. But the chain (middleware sets auth_context → full_access for empty role_map) is the load-bearing assumption; a future "no role map = least privilege" change would make the fail-open live and dangerous — which is precisely why the hardening is worth having even though it's currently a no-op.

WI-118 (trivial, correct; posture.py/certificates.py were already the reference pattern) and WI-082 (clean migration, no dead helper code remained) I'm fully confident in. WI-080: capChips verified alive, login testids fixed, cross-page non-collision confirmed via the dead-`api_keys.html` finding.

The thing I did *right* was running the full `-n auto` suite after the subagents finished, not just trusting their targeted runs. Every targeted run was green; the full run caught the `_store_sem` failure. Then confirming it was pre-existing via `git stash` (failed on clean HEAD too) was the move that let me distinguish "I broke it" from "it was already broken" — and the fix fell out of WI-092 for free.

## On the work done

**What went well:** The two-subagent adversarial review (kimi for security, glm for correctness) caught real issues that I would have missed. This is a recurring pattern: the unit suite passes but e2e catches integration issues. The AGENTS.md note about "a test you have never seen fail is a rumor" applies — the unit tests for user creation all used `role_id=None` explicitly, never the empty-string default.

## Gaps to flag

- **`scan.py:408`** — `_execute_deferred_post_commit` uses `contextlib.suppress(Exception)` which silently discards all webhook resolve failures. Should use targeted exception handling (SSRFBlockedError, TimeoutError, HTTPError) and log the rest. Flagged by the security reviewer.
- **`app.py:328-364`** — The scheduler's deferred webhook resolves execute sequentially in the scheduler thread. A slow PagerDuty API response stalls the scheduler. Should submit to the thread pool like event webhooks do.
- **`routes/api/hosts.py`** — `api_update_host_owner` still has the read-back query (`repo.get(host_id)`) inside the `get_write_lock()` block. It was supposed to be moved outside but the lock wraps the entire handler body. Minor performance issue.
- **Audit atomicity** — `record_audit()` uses a separate SQLite connection from the repository operations, so the audit log is not transactionally tied to the write it records. This is a pre-existing design constraint (the audit connection is separate by design), not a regression, but it means an action can be persisted without a corresponding audit trail if the audit insert fails.
- **No regression test for the `_validate_email` hardening** — the fix rejects newlines/semicolons/control chars, but no test exercises the new rejection paths. The existing test only checks comma rejection.
- **CI visual baselines** — the ubuntu-latest font drift will recur. No automated way to distinguish environmental drift from a real UI regression without the HTML-diff investigation I did this session.

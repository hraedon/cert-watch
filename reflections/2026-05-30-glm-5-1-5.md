## On the project

This is a well-organized codebase. The auth layer is clean — the Provider protocol makes it straightforward to add a new auth mechanism (LocalAdminProvider) and compose it with existing ones (_CompositeProvider). The Jinja2 template structure is reasonable; the macro extraction was mechanical once I identified the superset of icons across all 4 templates. The test infrastructure's `_reload_app` pattern is the one real pain point — it reloads modules globally but doesn't undo the side effects, causing test pollution across modules.

## On the work done

The BC-038 fix was clean — extract macro, import in each template, replace `{{ svg_icon() }}` with `{{ icons.svg_icon() }}`. Straightforward, all 330 tests passed immediately after.

The Plan 010 Slice 2 implementation went smoothly overall. The tricky part was the `isinstance` check for break-glass detection in `routes/auth.py` — when tests use `_reload_app` (which does `importlib.reload`), the reloaded module's `LocalAdminProvider` class becomes a different Python class object than the one imported at the top of the test file, so `isinstance()` returns False. I fixed this by using `type(auth).__name__` string comparison instead.

The auth test module's `_reload_app` is a pre-existing source of test pollution — it permanently mutates module globals (_signing_key, app.state) via importlib.reload. This causes ~20 failures in downstream test modules (dashboard, REST API) when run together. I filed BC-041 for this. Each module passes individually.

The `_scrypt_hash` function uses n=2^14 (16384), r=8, p=1 which is appropriate for a break-glass admin (not a high-frequency login path). The CLI command `cert-watch hash-password` prompts for password with getpass and confirmation.

## Gaps to flag

- `tests/test_auth.py:_reload_app` — Causes module state pollution for subsequent test modules. 20+ tests fail when entire suite is run together, but pass individually. See BC-041.
- `auth.py:481` — The `type().__name__` check for break-glass detection is fragile if someone renames `LocalAdminProvider` or `_CompositeProvider`. Consider a marker attribute (e.g., `is_break_glass: bool` on the provider) instead.
- `login.html:44-55` — The local admin form and the LDAP/primary form render identical HTML. If they need different styling or labels, the template should differentiate. Currently both say "Username" / "Password" / "Sign in" which is fine for break-glass but might confuse users who see it alongside an OAuth button.
- `__main__.py` — The `hash-password` subcommand uses `getpass` which doesn't work in non-interactive CI shells. Documentation should note this is interactive-only.

## On the work done

**Composite indexes (migration 0005):** Added `idx_cert_host_port_leaf` on `certificates(hostname, port, is_leaf)` and `idx_scan_history_host_port_ts` on `scan_history(hostname, port, scanned_at DESC)`. Also added to base schema for fresh DBs. Straightforward migration following the existing pattern.

**Rate-limit proxy bypass:** Replaced `request.client.host` with `_extract_client_ip(request)` in hosts.py (3 call sites), certificates.py (1 call site), and auth.py (1 audit call site). Also fixed `resolve_source_ip()` in audit.py to delegate to `_extract_client_ip()` — returns `None` instead of `"unknown"` when IP can't be determined (appropriate for audit logging). The subtlety here was that `_extract_client_ip` returns `"unknown"` as a sentinel for missing IPs (used in rate-limit keys), but audit logging should use `None`.

**_probe_hsts tests:** 10 unit tests covering non-443 port, HSTS present/absent, connection errors, SSL errors, pinned IP path (verifying correct IP and SNI), pinned IP SSL failure with socket cleanup, connection refused, and connection cleanup on success/error. The initial attempt to use `monkeypatch.setattr("cert_watch.scan.http.client.HTTPSConnection")` failed because `_probe_hsts` uses a local `import http.client` — the module-level attribute doesn't exist. Fixed by mocking `http.client.HTTPSConnection` directly (stdlib-level).

**CI fix:** TestOAuthJWKSVerification and TestJWKSCacheTTL always failed in CI (only `.[dev]` installed, not `.[auth-oauth]`). Added `@pytest.mark.skipif` class-level markers checking `importlib.util.find_spec("joserfc")` and `importlib.util.find_spec("authlib")`. CI is now green.

**Release workflow fix:** `release.yml` only bumped `deploy/k8s/kustomization.yaml`, leaving cert-watch-demo stuck on stale image tags. Fixed by looping over both `deploy/k8s` and `deploy/k8s-demo` directories, and adding `deploy/k8s-demo/kustomization.yaml` to `paths-ignore`.

**Version injection:** The UI showed "v0.3.0" (a stale hardcoded value). Replaced with build-time injection: Dockerfile accepts `GIT_TAG` and `GIT_COMMIT` build args, writes them to `_version.txt`. `__init__.py` reads this at import time, stripping any `v` prefix. The template shows `v{version} ({commit})`. Release workflow passes both args. Local dev falls back to the hardcoded defaults.

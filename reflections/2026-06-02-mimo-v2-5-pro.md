## On the project

The codebase is in strong shape for something that's grown to ~9,200 lines across 72 files. The architecture is well-decomposed — repository pattern, migration system, clean separation of routes/templates/data layer. The test suite (626 tests) is comprehensive and the CI pipeline catches regressions.

What impresses me most is the security posture. Multiple adversarial review rounds have left the SSRF protection, DNS-rebinding prevention, and auth layer in good shape. Parameterized SQL everywhere. Jinja2 auto-escaping on by default. The migration system is lightweight and idempotent.

What concerns me is the "wiring not connected" pattern that keeps appearing. Features get built (alert groups, drift detection, settings page) but the data flow between them has gaps: `extra_recipients` computed but never persisted, cascade deletes not updated when new tables are added, `_require_admin` checking auth but not authorization. This is exactly the kind of thing that happens when velocity outpaces integration testing.

## On the work done

The four bugs I fixed were all in the "wiring" category:

1. **BC-049** (DNS-rebinding bypass) — The `except OSError: pass` pattern in `_scan_host_once` was silently allowing scans to proceed without IP pinning. Clean fix: return `ScanError` instead. The existing tests needed `_resolve_host` mocking added since the DNS resolution now fails before the connection attempt.

2. **BC-048** (authorization bypass) — Added `CERT_WATCH_ADMINS` env var and updated `_require_admin` to check it. Break-glass local admin always has access. When `CERT_WATCH_ADMINS` is unset, backward-compatible behavior (any authenticated user). This is a pragmatic middle ground — full group/role-based admin checks would require storing groups in the session cookie, which is a bigger refactor.

3. **BC-051** (extra_recipients not persisted) — Created migration 0010, updated `SqliteAlertRepository.create()` to serialize `extra_recipients` as JSON, updated `_row_to_alert()` to deserialize. The `dict(row)` approach for `sqlite3.Row` objects was a small gotcha — `Row` doesn't have `.get()`.

4. **BC-050** (cascade deletes) — Added `cert_history` and `alert_group_certs` cleanup to both `delete_certificate_cascade()` and `SqliteHostRepository.delete()`. The cert_history delete uses fingerprint-based lookup since that table doesn't have cert_id.

I'm confident in all four fixes. The migration is idempotent (PRAGMA check before ALTER TABLE). The test updates are minimal and targeted.

## Gaps to flag

- **`settings.py:236`** — Error messages interpolated into redirect URLs without URL-encoding. LDAP server responses could contain characters that break the URL. Filed as BC-052.

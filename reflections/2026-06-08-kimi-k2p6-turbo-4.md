## On the project

The project is in a healthy state post-v0.7.0, but the CI revealed two brittle patterns that are worth calling out:

1. **SVG icons in JS strings are fragile.** The `icons.svg_icon()` Jinja macro renders multi-line `<svg>` tags with newlines. When embedded in a single-quoted JS string, this silently breaks the script and prevents the rest of the function from being defined. This happened in `dashboard.html` (line 763) and `certificate_detail.html` (lines 601–603). The fix is to add `|replace("\n", "")` to the icon call, but a better long-term fix would be to not render SVGs inline in JS at all — use a CSS class or data attribute that the JS sets, and have the SVG rendered via a template helper.

2. **mypy type drift in scan.py.** The `store_scan_posture` function expects `scanned_at: str | None`, but `scan.py` was passing `entry.scanned_at` (a `datetime`). This slipped through because the CI was already failing on the v0.7.0 release commit, so the mypy check never ran on the release code. The release was cut from a local dev branch where mypy wasn't enforced, and the CI was already red from the previous baseline.

## On the work done

**What went well:**
- The BC-123 refactor was clean — `tests/_helpers.py` with `seed_certificate()`, `seed_scanned()`, and `seed_host()` replaces 4 raw-SQL test helpers. The tests are more robust and less likely to break on schema changes.
- The E2E fix was satisfying because it required actual debugging (headless Playwright, CSP nonce check, script tag analysis) rather than just reading the error message.

**What was awkward:**
- The `tests/__init__.py` was missing, which meant `tests._helpers` was not importable. This is a common Python gotcha but it's not documented in the project's conventions.
- The `seed_certificate()` helper needed a `cert_id` override path for tests that assert on specific IDs (e.g., `test_renewal_window.py` asserts `cert_id == "stalled"`). The `SqliteCertificateRepository.add()` generates a UUID, so the helper needed a conditional path for deterministic IDs. This is fine but worth noting in the helper docstring.

**What I'd want a second pair of eyes on:**
- The `tags` column in `seed_certificate()` was not being set by the repository helper because `Certificate.tags` doesn't exist (tags are stored on the DB row, not the `Certificate` dataclass). I had to add a `tags` kwarg to the deterministic ID path. This is correct for the test helper, but it means the helper isn't a pure wrapper around the repository — it adds a capability the repository doesn't have.

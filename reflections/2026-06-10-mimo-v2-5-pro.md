# Session Reflection — 2026-06-10

**Work summary:** Closed 8 WIs (WI-007 through WI-017, excluding WI-006) across 4 commits. Implemented CT mis-issuance allowlist, Alertmanager resolve wiring, policy/posture dedup, SSE ping, and fixed 7 critical bugs found by adversarial review. Test count grew from 1703 to 1770.

---

## On the project

The codebase is in good shape. The policy engine (Plan 047 Workstream C) is mature enough that the remaining WIs were all small — fix-this-edge-case or add-this-test, not build-this-feature. The adversarial review pattern is paying for itself: this session found 7 critical/high bugs across 3 rounds, most of them things that would have been silent production failures (dead-code resolve path, auth escalation, deadlock, info-status suppression). The codebase conventions are well-documented in AGENTS.md and the agents mostly follow them.

## On the work done

The player-coach methodology (GLM + Kimi as implementers, adversarial reviewers to check) worked well this session. The reviewers consistently found real bugs that the implementers missed — not false positives, not style complaints. The most impactful finding was the `replace_scanned` alert-deletion ordering bug (WI-008): the feature was dead code in production because alerts were deleted before the resolve path could see them. Both the PagerDuty and Alertmanager resolve paths had this bug since Plan 022.

The one thing I'm less confident in: the WI-017 fix (RLock for policy concurrency) is correct but the merge-semantics change to PUT is a behavior change that could surprise operators who expected PUT to replace. The reviewer flagged this but I didn't address it — the merge behavior is the safer default for concurrent access, but it should be documented.

## Gaps to flag

- **`_issuer_matches` strict fallback uses CN extraction now, but the test data uses short strings, not real DNs.** A test with actual OpenSSL-formatted DNs (`/CN=R3/O=Let's Encrypt/C=US`) vs CT-formatted (`CN=R3, O=Let's Encrypt, C=US`) would lock in the DN-format fix. `tests/test_ct_monitor.py` — missing.
- **PUT /api/policy merge semantics aren't documented.** An operator who expects PUT to replace (REST convention) will be surprised that old rules persist. Either document it or add a `?mode=replace` query param. `routes/api/policy.py:129`.
- **`sse-starlette` was undeclared until this session.** Other undeclared transitive deps may exist — a `pip check` or `uv pip check` pass in CI would catch these.
- **The `acquire_policy_lock` + `save_policy_set_locked` pattern has no grep-ability guard.** A future contributor calling `save_policy_set` inside the lock will deadlock silently. Consider a linter rule or a `# NEVER call save_policy_set inside acquire_policy_lock` comment. `policy.py:415-427`.
- **The duplicate WIs (WI-009/010/012/013) were closed without investigation.** They appeared to be duplicates of WI-011 and WI-013, but I didn't verify — I closed them based on title matching. If any were tracking distinct issues, they're now lost.

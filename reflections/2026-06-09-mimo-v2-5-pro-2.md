## On the work done

BC-156 (StartTLS TOFU) was the most interesting fix. The `ldap3` library exposes `conn.socket` as an `ssl.SSLSocket` after `start_tls()`, which made the implementation cleaner than I expected — no raw LDAP protocol bytes needed. The fallback to `_probe_tls_chain` on the same port is pragmatic but not guaranteed to work (some servers present different certs on StartTLS vs LDAPS). The implementation documents this limitation.

BC-163 (exception narrowing) was more mechanical but arguably higher-impact for code health. The pattern of catching bare `Exception` in security-critical auth paths was hiding real bugs. The changes are conservative — I kept `except Exception` as a final safety net in ldap_provider.py and oauth_provider.py's ID token retry path, with `# noqa: BLE001` to document intent.

The duplicate cleanup (BC-158/BC-154) was trivial but valuable — stale duplicates in the backlog are noise.

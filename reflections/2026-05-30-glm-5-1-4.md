## On the work done

Plan 010 Slice 1 went smoothly. The key design decision was keeping the authorization gate at login/callback time only, not in the middleware â€” this means authenticated sessions aren't re-checked on every request, which is correct for role/group membership that changes rarely and can be enforced at login. The `AuthResult.groups`/`roles` fields are populated by future provider slices (LDAP group filters, OAuth roles claims) but the gate infrastructure is provider-agnostic and ready.

The `read_secret()` helper is simple and correct. The `or ""` pattern for `ldap_bind_password` and `oauth_client_secret` preserves existing behavior (empty string = not set) while `smtp_password` remains `None`-able because `build_alert_config()` uses `None` to detect "SMTP not configured."

The "Download PEM" fix was a real gap â€” the button linked to the JSON endpoint, which is wrong UX. The new `/api/certificates/{id}/pem` endpoint returns proper PEM bytes with `Content-Disposition: attachment`.

The SHA-1 signature test couldn't create actual SHA-1-signed certificates (modern `cryptography` refuses), so I verified the OID constants exist and the detection code references them. This is an acceptable trade-off â€” the OID comparison path is straightforward and can't silently break. The TLSFeature (OCSP Must-Staple) tests needed the correct import path (`cryptography.x509.TLSFeatureType`, not `.oid.TLSFeatureType`).

## On the project

The codebase is well-structured with clean separation between modules. The plan documents (007-011) are excellent — precise phase boundaries, explicit file paths, clear "what we're not doing" sections. This made implementation straightforward.

The test infrastructure is the weakest point. The `importlib.reload()` pattern used by `_reload_app` to create per-test app instances is fragile — it mutates module globals, causes class-identity drift when modules reference each other's classes (middleware → auth), and required an autouse save/restore fixture as a band-aid. A proper dependency-injection approach (passing `settings` and `auth_provider` to the app factory) would be cleaner but is a larger refactor.

## On the work done

BC-041 was the most interesting fix. The root cause — `isinstance(auth, NoAuthProvider)` returning `False` because `importlib.reload()` created a new `NoAuthProvider` class that middleware didn't reference — was non-obvious. I initially tried restoring just `config` and `app` module dicts, which didn't fix it. Adding `auth` to the restore set was the key. The shared `reload_app` conftest fixture is a real improvement — 7 copies eliminated.

Plan 010 Slice 3 (LDAPS) went smoothly. The `ldap3` API for ServerPool, Tls, and FIRST strategy is well-documented. The transitive group filter using OID 1.2.840.113556.1.4.1941 is a one-liner in the search filter. I'm confident in this implementation.

Plan 006 Phase 3 (CT reconciliation) is straightforward — query crt.sh, compare hostnames against tracked hosts, compute coverage. The `ReconciliationResult` dataclass keeps the API clean. Coverage percentage calculation is defensible.

The operator runbook (`docs/runbook.md`) is comprehensive. It covers the full lifecycle: deploy, upgrade, backup, restore, troubleshooting, config reference, auth wiring, scale ceiling, and the `/metrics` exposure decision.

## Gaps to flag

- **`tests/conftest.py:_isolated_data_dir` does `importlib.reload(_config)` but NOT `importlib.reload(app)`** — the `reload_app` fixture does both. Tests that only need config (no app) still get a config reload from the autouse fixture, which is redundant but harmless. Worth noting for anyone debugging test setup.
- **The LDAPS `_build_tls()` method checks `is_ldaps` with `any(s.lower().startswith("ldaps://") for s in server_urls)`** — if someone mixes `ldap://` and `ldaps://` URLs in the comma-separated list, the SSL/TLS configuration will be set for LDAPS but applied to all servers in the pool. ldap3 handles this per-Server, but the `use_ssl` flag in `authenticate()` is computed globally. This edge case is unlikely but worth knowing about.
- **CT reconciliation doesn't cache crt.sh results** — each call hits the network. For the API endpoint this means repeated calls trigger repeated network requests. Should be fine for periodic scheduler use but could be slow if called frequently.
- **The runbook mentions `CERT_WATCH_COOKIE_SECURE`** which comes from `middleware.py:_COOKIE_SECURE = os.environ.get("CERT_WATCH_COOKIE_SECURE", "1")`. This env var works but isn't in the README config table. Minor doc gap.

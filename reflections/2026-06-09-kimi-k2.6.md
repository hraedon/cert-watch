Implemented Plan 040 foundation: `users`/`roles` DB tables (migration 0019), repositories, `LocalAdminProvider` DB auth, session token email field, settings UI tabs for role/user CRUD + LDAP group→role mapping. 17 new tests; 1415 total pass.

## On the project

cert-watch is a well-scaffolded FastAPI app with strong conventions (migrations, repositories, CSP nonces, BC-NNN tracking). The auth decomposition (Plan 021) and RBAC skeleton (Plan 035) made the roles/users feature a natural extension rather than a hack. The template duplication between `settings_page`, `roles_page`, and `users_page` (each manually passing `auth`, `smtp`, `alert`, `env_overrides`) is a small but real friction point — a shared settings-context builder would help.

## On the work done

The SMTP double-send bug was a clean one-line fix (remove two redundant `addEventListener` calls in settings.html). The roles/users work was larger but followed existing patterns, so it landed cleanly. The session token format extension to 7 parts (adding `email`) was done with backward compatibility in mind — old 3/4/6-part tokens still decode. One subtle bug: `SqliteRoleRepository` wasn't imported inside `LocalAdminProvider.authenticate()`, causing a `NameError` that was silently swallowed by the broad `except Exception` fallback — fixed after a debug session. I'd want a second pair of eyes on the `_dummy_verify` signature change (now takes `hash_ref`) to confirm no other timing paths were affected.

## Gaps to flag

- `src/cert_watch/routes/settings.py:roles_page` and `users_page` duplicate context setup (empty `auth`/`smtp`/`alert`/`env_overrides` dicts) — a shared helper would reduce drift.
- `tests/test_no_inline_styles.py` — budget for `settings.html` raised from 3→5 due to `style="display:inline"` on delete forms. Convert to a CSS class when convenient.
- No E2E coverage for the new roles/users UI tabs (only unit tests in `test_users_roles.py`).
- The LDAP role mapping form (`settings.html`) uses `role_map_{{ r.name }}` as the input name — if a role name contains spaces or special chars, the form key may not round-trip correctly. Should use `role_map_{{ r.id }}` instead.
- `middleware.py:get_auth_context()` does a DB lookup on every request to resolve email when the session token lacks it. This is fine for low-traffic use but should be revisited if load grows (cache the user lookup for the request lifetime, or ensure the token always carries email after login).

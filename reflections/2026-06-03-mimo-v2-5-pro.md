## On the project

cert-watch is a mature, security-conscious TLS certificate monitor. The codebase is clean — no TODO/FIXME/HACK comments, no hardcoded secrets, no debug flags. Security hardening is genuinely thorough (SSRF protection, CSP nonces, CSRF double-submit, session revocation, secure-by-default auto-provisioning). The spec-driven approach works: work items have clear acceptance criteria, and the implementation matches them.

The `queries.py` file at 2,378 lines is the main structural concern (BC-094). It bundles dashboard queries, aggregate helpers, encryption utilities, and kv_store operations. It's not blocking but it's real tech debt.

The deployment story is solid — k8s with Argo CD GitOps, compose, systemd, and IIS all documented. The demo secret.yaml was the one real deploy hazard: a placeholder secret with no auth-secret that could be blindly applied.

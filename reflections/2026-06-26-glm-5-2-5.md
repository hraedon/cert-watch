## On the project

The template architecture is clean given its constraints — all JS is inline behind CSP nonces, all CSS lives in one 1260-line tokens.css file, and the `cw-*` vocabulary is extensive enough to cover most needs. The main weakness is the same one this PR addressed: without shared macros, every template reinvents the same patterns (pill rendering, tone mapping, stat cards) and they drift. The `macros/icons.html` pattern was the right precedent, and `macros/urgency.html` follows it. The project is in maintenance mode, so the bar for changes is "does it fix a real problem without breaking anything" — this PR clears that bar.

## On the work done

**Confident in:** The urgency_pill macro extraction and filter additions. These are the keystone — the threshold logic now lives in exactly one place (filters.py), the labels in one dict, and the tone mapping in one function. The JS cwUrgency() is gone. The pivot API returns urgency+label so JS doesn't duplicate logic. Tests added for all new functions.

**Confident in:** The report page migrations. The `rdy-*`/`cmp-*`/`cry-*` vocabularies were genuine reinvention — they all had the same stat-box/table/chip/empty patterns that `cw-*` already covers. The rewrites are cleaner and use the established component vocabulary. The inline-style budgets dropped significantly (compliance 6→1, readiness 5→0).

**Less confident in:** The ARIA implementation is correct in structure but incomplete in behavior. The reports dropdown has `role="menu"`/`role="menuitem"` but the menu items aren't keyboard-focusable (no tabindex, no roving tabindex, no arrow-key nav). The insights tabs use `role="tab"` on `<a>` elements which works for navigation but doesn't implement the full WAI-ARIA tab keyboard pattern. The focus trap in the slide-over is basic (cycles first/last focusable) but doesn't capture links or custom controls. These are all better than no ARIA, but they're not complete WAI-ARIA implementations.

**The `aria-hidden="true"` on dot elements** was a consistency sweep — the dots are decorative and the text label is the real a11y cue. This was low-risk and high-consistency.

## On what remains

1. **Visual baselines need regeneration** — the dashboard, certificate_detail, compliance, readiness, and crypto templates all changed visually. The e2e visual regression job will fail on push. The baselines must be regenerated from the CI artifact (per AGENTS.md instructions), not locally.
2. **Reports dropdown keyboard accessibility** — the `role="menu"`/`role="menuitem"` needs tabindex + arrow-key nav to be a complete implementation. Currently it's click-only with Escape-to-close.
3. **Insights tab keyboard pattern** — the tabs work via navigation (anchor links with `?tab=`) but don't implement arrow-key tab switching. This is acceptable for navigation-based tabs but is not a full WAI-ARIA tab pattern.
4. **scan_history, host_detail, audit pills** — these still hand-write `cw-pill` blocks (with custom labels like "Scan failed", "Pending", action names). They're not urgency-based so they don't use the threshold logic, but they could adopt the macro for consistency if it accepted a custom label parameter.

## Gaps to flag

- **`certificate_detail.html:109,113`** — chain status chips use Unicode emoji (⚠, ✗) that are not `aria-hidden`. These are decorative and should be hidden from screen readers.
- **`dashboard.html` Reports menu** — JS-built menu items (`<div role="menuitem">`) have no `tabindex`, so they're invisible to keyboard users. Only the trigger button and Escape-to-close work.
- **No tests for keyboard behavior** — the Enter/Space activation on `[data-action]` rows, Escape-to-close on slide-over, and focus trap have no unit or e2e tests. The functionality works manually but could regress silently.
- **`base.html:260-268`** — the synthetic `el.click()` for keyboard activation could double-fire on native anchors with `data-action`. Currently safe because no anchors use `data-action`, but it's a latent footgun.
- **Visual e2e baselines** — not regenerated. CI will fail on the visual job. This is expected and documented in AGENTS.md.

## On the project

The breadcrumb system is genuinely excellent — it's the reason this project can sustain multi-session work without losing context. Every issue found in the adversarial review had a clear paper trail. But there's an accumulation problem: two review rounds found 27 issues total, and the security layer is accreting rather than consolidating. Every fix adds another `if` check, another env var, another helper function. The next review round will find fewer issues because the surface is being hardened, but what remains needs structural consolidation rather than point fixes.

`auth.py` at 877 lines is the project's biggest risk concentration. It does session management, LDAP with TLS config, OAuth with JWKS caching, scrypt hashing, and provider assembly — six distinct security concerns in one file. Any one of those concerns having a bug requires understanding all six. That's why Plan 021 exists.

## On the work done

The adversarial review was the most valuable part of this session. Running four parallel agent tasks (auth, routes, scan, DB/templates) and then triaging the combined 27 findings down to the 15 I fixed was efficient. But I should note: the agents found issues I would have missed in a linear read-through, particularly the DNS parser infinite loop (not a security issue I'd think to check) and the OAuth state bypass when state is empty string (the `if state:` pattern looked correct until you think about the empty string case).

The HSTS probe fix was more involved than expected — `_probe_hsts` needed a `pinned_ip` parameter and a `set_tunnel()` call for HTTPS. The original code just used `HTTPSConnection(hostname)` which re-resolved DNS. The fix connects to the pinned IP with the hostname as SNI/Host. This is the same pattern as the TLS scan path, which is reassuring — the SSRF protection is now consistent across all connection types.

The `build_auth_provider` ValueError change is the one I'm least confident about. Silently degrading to NoAuthProvider was definitely wrong (security vulnerability), but raising ValueError means misconfigured deployments crash at startup instead of running open. That's the right trade-off, but it's a behavior change that could surprise operators who had `AUTH_PROVIDER=ldap` set without `LDAP_SERVER`. I updated the tests, but operators upgrading will need to either fix their config or explicitly set `AUTH_PROVIDER=none`.

## On what remains

**Before shipping (Plan 020 S1):** The `Depends(require_auth)` sweep eliminates the class of bug that caused 4 auth bypasses. This is the single highest-value remaining change.

**Before shipping (Plan 021 S1–S7):** Auth decomposition makes future security review tractable. Each module (session, LDAP, OAuth, local admin) can be reviewed in isolation.

**Before shipping (Plan 020 S4):** CSP nonces remove `'unsafe-inline'` from script-src, which currently allows any XSS that gets past `escHtml()`. The urgency whitelist helps, but nonces are the proper fix.

**Nice to have:** Plan 020 S2 (rate-limit dependency), S3 (audit side-effect), S5 (get_db dependency) are mechanical and can be done any time.

## Gaps to flag

- **BC-070**: `check_csrf()` accepts `_csrf_token` in query parameters. Query-param tokens leak to logs and Referer headers. Remove the fallback on state-changing routes. (middleware.py:235)
- **BC-071**: OAuth userinfo fallback has no nonce binding. An attacker with any access token can authenticate. (auth.py:619–630)
- **BC-072**: `_CompositeProvider` timing leak on local admin username. Immediate return on username mismatch vs ~100ms scrypt on match. (auth.py:769–773)
- **BC-073**: `_list_unified_entries_raw()` loads the full certificates table into memory. At fleet scale this will OOM. Covered by Plan 018 B2. (queries.py:593–615)
- **BC-074**: Ephemeral signing keys invalidate all sessions on restart and don't share across workers. Covered by Plan 018 B1 (SecurityContext). (auth.py:30–36)
- **`_load_unified_filtered()` uses f-string SQL**: The `host_where` parameter is only passed from internal callers with hardcoded values, but it's a latent injection surface. A comment is there; a whitelist would be better. (queries.py:618–680)
- **`escHtml()` doesn't escape quotes**: Safe in current usage (CSS classes and text content) but fragile if future code puts values in HTML attributes. (dashboard.html:563–567)
- **Rate-limit IP extraction inconsistency**: Fixed in this session (views.py and auth.py now use `_extract_client_ip`), but the pattern is fragile — each new rate-limited route must remember to use it. Plan 020 S2 fixes this structurally.

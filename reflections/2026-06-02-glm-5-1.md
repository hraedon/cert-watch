## On the project

The codebase has grown significantly (~9,200 lines in 72 files since last push) and the quality is generally high — 626 tests pass, lint is clean, and the architecture is well-decomposed. However, the rapid feature velocity has left some "wiring not connected" issues: `extra_recipients` is computed but never persisted, `_require_admin` checks auth but not authorization, and cascade deletes weren't updated when new tables were added. These are exactly the kind of gaps a focused review catches.

The settings page (Plan 019) is the riskiest new surface — it stores LDAP bind passwords and OAuth client secrets in plaintext in SQLite `kv_store`. Combined with the authorization bypass in `_require_admin`, any authenticated user can extract service account credentials. This needs priority attention before the push.

## On the work done

The three-agent parallel review strategy worked well for coverage. The security agent caught the `_require_admin` issue and SSRF vectors; the data-layer agent caught the `extra_recipients` gap and cascade deletes; the feature agent caught the DNS-rebinding bypass and drift masking.

Fixes I'm confident in: `__import__` → proper import, temp file leak, CRLF sanitization, json.loads guard, CSRF token `|tojson`, webhook URL validation, drift detection unmasking.

The dead ternary in views.py was a false alarm — the pivot view loads entries via AJAX, so `all_entries: page_entries` is correct (both branches intentionally return `page_entries`). I reverted that change after the test failure caught it.

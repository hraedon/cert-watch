## On the work done

The pytest-cov addition was straightforward — the 70% baseline is conservative (actual: 82%) and the omit rules for `__main__` and migrations are appropriate.

The E2E tests were trickier. The settings test (`test_settings_flows.py`) uses the existing `cert_watch_server` fixture which runs with `ALLOW_UNAUTH=1`, so it can test form round-trips without auth. The auth test (`test_auth_flows.py`) needed a separate `auth_server` module-scoped fixture that runs without `ALLOW_UNAUTH` on loopback, triggering the setup wizard. The test exercises the full setup -> login -> logout -> API-auth gate flow. One thing I'm confident about: the setup wizard E2E test is the right approach rather than trying to set `CERT_WATCH_LOCAL_ADMIN_*` env vars, because it tests the actual user-facing flow.

The key rotation work surfaced a design choice: `fernet_decrypt` returns the raw `enc:v1:` blob on failure (line 46 of queries.py) rather than raising. This is a silent degradation — the startup warning I added catches this at boot time, but any runtime read of a corrupted value still silently returns garbage. This is acceptable for now because it only happens after a key rotation error, and the affected keys are all in `_SENSITIVE` (config values read at startup, not hot paths).

The `re_encrypt_kv_store` function does individual `UPDATE` + `commit` per value rather than batching. At the scale this runs (typically 2-4 sensitive kv_store values), this is fine. If someone has hundreds of alert-group secrets, it would be worth batching, but that's not a realistic scenario.

## Gaps to flag

- **Auth E2E tests need Playwright installation to run.** The `test_auth_flows.py` module-scoped fixture starts a real server process. CI needs `playwright install --with-deps chromium` and the E2E marker activation (`pytest -m e2e`). The existing CI workflow already does this.
- **`fernet_decrypt` returns raw ciphertext on failure** (`queries.py:46-47`). This is intentional (backward compat with plaintext values) but means runtime reads of encrypted-but-corrupt values silently return `enc:v1:...` garbage. Not a real risk since the startup warning flags it, but worth noting.
- **The auth E2E test reuses the `auth_server` fixture across test methods.** If one test fails or the server crashes, subsequent tests in the class will also fail. This is standard for Playwright fixtures but worth knowing if debugging CI failures.
